# Sheratan Work Orders Pack

Enthält 5 Work Orders (0001–0005) als klaren, agenten-tauglichen Sprint für:
- Repo-Map & Status-Klarheit
- README Redirects für deprecated Repos
- Core/Hub Boundary Definition
- Namenskonvention & Migrationsplan
- Business/Ops Runbook (Ist-Stand)

## Verwendung (empfohlen)
1. Bearbeite immer **nur eine** Work Order gleichzeitig.
2. Nutze DoD/Constraints als harte Leitplanken.
3. Review → Commit/PR → erst dann nächste Work Order.
